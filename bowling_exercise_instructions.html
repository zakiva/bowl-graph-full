    • Gutter ball detection: if the ball leaves the lane edges it falls into the gutter and knocks down zero pins

  3. Pin collision and toppling:
    • Ball-to-pin collision detection (sphere vs. pin bounding cylinder)
    • Basic pin-to-pin propagation (a falling pin can knock neighbours)
    • Knocked pins visibly topple over (rotate and fall), then are removed / marked as down
    • The set of standing pins is tracked accurately
  4. A ten-frame bowling scoring system:
    • 10 frames, two rolls each (three in the 10th on a strike/spare)
    • Correct strike (X), spare (/), and open-frame scoring with the proper bonus rules

    • Pins reset between rolls / frames as appropriate
  5. Game flow and state management:
    • Detect the end of a roll (ball stopped or left the lane), count the fallen pins, update the score, and advance to the next roll/frame
    • Reset the ball to the approach for the next roll
    • 'R' resets the pins / starts a new game
    • Clearly indicate the game is over after the 10th frame

Note: Use simplified, hand-written physics only. Do NOT add an external physics engine (cannon-es, ammo.js, etc.). The point is to implement the motion and collision yourself.

SPECIFIC IMPLEMENTATION DETAILS

1. Aiming & Power Controls

  • While aiming, let the player move the ball left/right across the foul line (and optionally rotate the launch direction) using the arrow keys.
  • Pressing Space starts the power meter, which sweeps back and forth between min and max. Pressing Space again locks the power and releases the ball with a velocity proportional to that power.
  • Model the game as a small state machine, e.g. aiming → power → rolling → resolving → (next roll). Only accept aim/power input in the matching state.
  • Disable orbit input while the ball is rolling if it interferes with play (the 'O' toggle must still function).

2. Ball Physics (Simplified)

  • Give the ball a velocity vector at release. Each frame, compute deltaTime and advance the ball: position += velocity * deltaTime. A THREE.Clock (clock.getDelta()) is convenient.
  • Optionally apply a small sideways acceleration for spin/curve (a "hook") and a little rolling friction so the ball decelerates.
  • Gutter detection: if the ball's X position passes the lane edge (|x| > half-lane-width), drop it into the gutter and treat the roll as 0 pins.
  • End the roll when the ball reaches the pin deck, leaves the lane, or effectively stops.

3. Pin Collision & Toppling

  • Approximate each pin as an upright cylinder. A ball-pin hit occurs when the horizontal distance between the ball centre and the pin axis is less than (ball radius + pin radius).
  • When a pin is hit, mark it as falling and animate it toppling — rotate it about a horizontal axis pointing away from the impact direction until it lies flat, then remove it or flag it as down.
  • For pin-to-pin propagation, a falling pin that is close enough to a standing neighbour can knock that neighbour down too. A simple distance check between pin positions is enough.
  • Keep a boolean (or list) of which pins are still standing; the number that fell during the roll is what feeds the scorecard.

4. Ten-Frame Scoring

  • Track 10 frames. Each frame normally has up to two rolls; the 10th frame allows a third roll if the player throws a strike or spare.
  • Scoring rules:
    • Strike (all 10 on the first roll): 10 + the pins from the next two rolls.
    • Spare (all 10 across two rolls): 10 + the pins from the next one roll.
    • Open frame: just the pins knocked down.
  • Display the per-frame rolls with standard notation (X for a strike, / for a spare) and a running cumulative total in the scorecard container you created in HW05.
  • A perfect game (twelve strikes) totals 300 — useful for sanity-checking.

5. Game Flow

  • After each roll resolves: count fallen pins, update the scorecard, decide whether the frame continues or advances, reset the standing pins when a new frame begins, and return the ball to the approach for aiming.
  • Provide an R control to reset the pins / start a fresh game, and clearly indicate when the game is finished.
